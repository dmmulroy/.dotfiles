# Goal
Reduce repeated Helium remote-debugging permission prompts during authenticated `webfetch` by caching auth state per selected Helium profile / agent session instead of per URL.

## Context
Current behavior was improved to cache auth resolution per URL within a single `webfetch` execution, which prevents duplicate prompts from the Cloudflare fallback retry path.

However, repeated `webfetch` calls can still reconnect to CDP and trigger the Helium remote-debugging prompt again.

## Desired behavior
- Cache authenticated cookie source state per selected Helium profile for the current agent session.
- Reuse the cached cookie jar across multiple `webfetch` calls.
- Still perform URL-specific cookie filtering per request via existing selection logic.
- Keep product constraints unchanged:
  - no browser launch
  - `webfetch` remains the only fetch tool
  - `/web-profile` remains the selector
  - footer-only status

## Suggested design
1. Add an in-memory auth cache keyed by selected Helium profile identity.
   - key could include `userDataDir + profileId`
2. Cache the raw cookie jar / auth-source result, not the final `Cookie` header.
3. On each request:
   - reuse cached cookies
   - run `selectCookiesForUrl(...)` for the target URL
4. Add cache invalidation rules:
   - profile switch
   - session start/reload
   - TTL expiry
   - explicit auth-source failure
5. Choose a conservative TTL (for example 30–120s) so auth state stays reasonably fresh.
6. Consider whether CDP failures should also be cached briefly to avoid immediate re-prompt loops.

## Open question
- Decide whether to also support an optional disk-cookies-only mode in the future for users who want to avoid CDP prompts entirely.

## Likely files
- `agent/extensions/web-tools/index.ts`
- `agent/extensions/web-tools/auth.ts`
- `agent/extensions/web-tools/webfetch.ts`
- maybe a new small cache module under `agent/extensions/web-tools/`
- tests in:
  - `agent/extensions/web-tools/test/auth.test.ts`
  - `agent/extensions/web-tools/test/webfetch.test.ts`

## Acceptance criteria
- Multiple authenticated `webfetch` calls against the same selected Helium profile do not reconnect to CDP on every call within the cache TTL.
- URL/path/domain-specific cookie filtering still works correctly.
- Cache is invalidated on profile change and expiry.
- Tests cover cache reuse and invalidation.
