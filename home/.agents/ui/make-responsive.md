# Make Responsive

Use this when the user wants an existing desktop-oriented UI to work well across mobile, tablet, and desktop breakpoints.

## Activation

Activate when the user asks to:

- make an existing UI responsive
- fix mobile, tablet, or breakpoint-specific layout issues
- adapt a desktop-only design to smaller screens
- resolve overflow, wrapping, clipping, or cramped layout problems on narrow viewports

Do not activate when:

- the user wants a brand-new design or layout
- the request is only about code structure or component extraction
- the request is only about dark mode or standalone image adaptation

## User-facing progress updates

Keep the user informed so longer runs do not look stuck.

- One-line status update before each major phase.
- Concrete and lightweight: what you are doing now, not verbose logs.

## Rules

- Read and follow the [Responsive Design](uidotsh://ui/design-guidelines/responsive-design) rules before making changes
