Audit complete.

Findings:
- `nvim-treesitter` main is a full incompatible rewrite for Neovim 0.12+.
- Current config in `lua/plugins/treesitter.lua` relies on legacy `require('nvim-treesitter.configs').setup(...)` API and lazy-loading, both incompatible with main.
- `nvim-treesitter` main only retains a minimal `require('nvim-treesitter').setup({ install_dir = ... })` config surface; highlighting/folds are now core Neovim, indentation is enabled via `indentexpr`, parser installation is explicit via `install()`/`:TSUpdate`.
- Current `ensure_installed`, `sync_install`, `highlight`, `indent`, `autopairs`, `autotag`, `incremental_selection`, and nested `textobjects` config will need migration/removal.
- `nvim-treesitter` main explicitly does not support lazy-loading; current `event = { 'BufEnter' }` should become eager load.
- `nvim-treesitter-textobjects` main requires direct `require('nvim-treesitter-textobjects').setup(...)` plus manual keymaps; legacy nested textobjects config under treesitter will not be consumed.
- `nvim-ts-autotag` should be explicitly configured via `require('nvim-ts-autotag').setup({...})`; current implicit legacy integration via `nvim-treesitter.configs`/`define_modules` is deprecated and likely to break with treesitter main.
- `nvim-treesitter-context` upstream default branch is still `master`; current config looks compatible and likely does not need migration.
- Current parser language list appears still supported on `nvim-treesitter` main; no parser renames found for the configured languages.
